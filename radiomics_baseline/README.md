# Radiomics Baseline for pCR Prediction

This directory implements a radiomics pipeline for predicting pathological complete response (pCR) from DCE-MRI breast cancer imaging. It extracts PyRadiomics features from multi-phase DCE-MRI images and tumor segmentations, trains classifiers (logistic regression, Random Forest, XGBoost), and evaluates performance via stratified 5-fold cross-validation.

## Data

### Source Data
- **Images**: DCE-MRI NIfTI files from the MAMA-MIA dataset, stored on the cluster at `/net/projects2/vanguard/MAMA-MIA-syn60868042/images/`
- **Masks**: Expert tumor segmentations at `/net/projects2/vanguard/MAMA-MIA-syn60868042/segmentations/expert/`

### Labels and Splits
- **labels.csv**: Contains `patient_id`, `pcr` (0/1), `tumor_subtype`, and `site` for ~1500 patients across 4 sites (DUKE, ISPY1, ISPY2, NACT). Generated from source JSON via `scripts/make_labels_from_json.py`.
- **splits_train_test_ready.csv**: Pre-computed train/test split with columns `patient_id` and `split` (values: "train" or "test"). This file was manually curated to ensure balanced site/subtype representation. It is not generated by any script in the repo.
- Cross-validation (5-fold, stratified by pCR label) is performed **within the training set only** using `StratifiedKFold(n_splits=5, shuffle=True, random_state=42)`.

## Pipeline Overview

The pipeline has two main stages:

### Stage 1: Feature Extraction (`radiomics_extract.py`)
Extracts PyRadiomics features (firstorder, shape, GLCM, GLRLM, GLSZM, GLDM, NGTDM) from DCE-MRI images. Supports:
- Multi-phase extraction (e.g., pre-contrast + post-contrast)
- Peritumoral region analysis (2D in-plane or 3D isotropic dilation)
- Kinetic parameter maps as additional image inputs
- Per-patient checkpoint caching for resumable extraction

### Stage 2: Model Training (`radiomics_train.py`)
Trains and evaluates classifiers on extracted features. Supports:
- Classifiers: logistic regression (elastic net), Random Forest, XGBoost
- Feature selection: correlation pruning, kBest (ANOVA), mRMR
- ComBat harmonization (parametric/non-parametric, fold-safe)
- CV-only mode for leakage-safe evaluation
- Per-subtype and per-site metrics

### Kinetic Maps (`generate_kinetic_maps.py`)
Generates voxel-wise kinetic parameters from multi-phase DCE-MRI:
- Enhancement ratios: E_early, E_peak
- Slopes: slope_in, slope_out
- Area under curve: AUC
- Optional: wash_in/wash_out subtraction maps, time-to-peak

## How to Run

### Environment Setup
```bash
micromamba activate vanguard
cd radiomics_baseline
```

Required packages: `pyradiomics`, `scikit-learn`, `pandas`, `numpy`, `matplotlib`, `SimpleITK`, `joblib`. Optional: `mrmr-selection`, `xgboost`, `neuroCombat`.

### Option A: Run a Single Experiment from YAML Config
```bash
python scripts/run_experiment.py configs/exp_peri5_multiphase_logreg.yaml
```
This runs extraction (if not cached) followed by training, writing results to the `outputs/` directory specified in the config.

### Option B: Run an Ablation Sweep
```bash
python scripts/run_ablations.py configs/sweep_2d_vs_3d.yaml \
  --generated-dir configs/generated
```
This generates individual experiment configs from the sweep grid and runs them sequentially. Experiments sharing the same extraction parameters automatically share cached features.

### Option C: Run Stages Independently
```bash
# Stage 1: Extract features
python scripts/radiomics_extract.py \
  --images /net/projects2/vanguard/MAMA-MIA-syn60868042/images \
  --masks /net/projects2/vanguard/MAMA-MIA-syn60868042/segmentations/expert \
  --labels labels.csv \
  --splits splits_train_test_ready.csv \
  --output outputs/my_extraction \
  --params pyradiomics_params.yaml \
  --image-pattern "{pid}/{pid}_0001.nii.gz,{pid}/{pid}_0002.nii.gz" \
  --mask-pattern "{pid}.nii.gz" \
  --peri-radius-mm 5 --peri-mode 3d --n-jobs 8

# Stage 2: Train model
python scripts/radiomics_train.py \
  --train-features outputs/my_extraction/features_train_final.csv \
  --test-features outputs/my_extraction/features_test_final.csv \
  --labels labels.csv \
  --output outputs/my_model/training \
  --classifier logistic --logreg-penalty elasticnet --logreg-l1-ratio 0.5 \
  --corr-threshold 0.9 --k-best 50 --feature-selection kbest \
  --grid-search --cv-folds 5 --cv-only
```

### Generating Kinetic Maps
Kinetic maps must be generated before running extraction with kinetic features:
```bash
sbatch scripts/slurm_generate_kinetic_maps.sh
```

### Running on SLURM
All SLURM scripts are in `scripts/`. Submit individual sweeps or use the master script:
```bash
bash scripts/submit_all.sh          # submit all sweeps
bash scripts/submit_all.sh --dry    # preview commands without submitting
```

## Output Structure

Each experiment produces:
```
outputs/<experiment_name>/
├── extraction/                   # Cached extracted features
│   ├── features_train_final.csv
│   └── features_test_final.csv
├── training/
│   ├── metrics.json              # Test set metrics
│   ├── predictions.csv           # Test set predictions
│   ├── model.pkl                 # Trained model
│   ├── plots/                    # ROC, PR, calibration curves
│   └── cv/
│       ├── metrics.json          # CV metrics (mean + std AUC)
│       ├── metrics_per_fold.json # Per-fold breakdown
│       ├── predictions.csv       # Out-of-fold predictions
│       └── plots/
└── run_config.yaml               # Snapshot of config used
```

Use `cv/metrics.json` for model ranking (primary: `auc.mean`, secondary: `auc.std`).

## Configuration

### Base Experiment Configs (`configs/exp_*.yaml`)
Each config has three sections:
- **paths**: Data locations (images, masks, labels, splits, output directory). Cluster data paths (images, masks) must be updated for your environment; other paths are repo-relative.
- **extract**: Extraction parameters (image patterns, peritumor settings, binning)
- **train**: Training parameters (classifier, feature selection, CV settings, harmonization)

### Sweep Configs (`configs/sweep_*.yaml`)
Define a `base_config` and a `sweep` block with parameters to vary. `run_ablations.py` generates the Cartesian product of all sweep parameters.

### Available Configs
| Config | Description |
|--------|-------------|
| `exp_peri5_multiphase_logreg.yaml` | Baseline: 5mm peritumor, 2-phase DCE, logistic regression |
| `exp_kinetic_maps.yaml` | Raw phases + kinetic parameter maps |
| `exp_combat_matrix_*.yaml` | ComBat harmonization experiments |
| `sweep_phases.yaml` | Peri radius x image phases x classifier (24 configs) |
| `sweep_2d_vs_3d.yaml` | 2D vs 3D peritumor ring comparison (24 configs) |
| `sweep_kinetic_maps.yaml` | Image type comparison: raw, kinetic, subtraction (24 configs) |
| `sweep_feature_selection.yaml` | kBest vs mRMR feature selection (6 configs) |
| `sweep_concat_vs_aggregate.yaml` | Feature concatenation vs aggregation (2 configs) |
| `sweep_sites.yaml` | Per-site models (4 configs) |
| `sweep_test_subtypes.yaml` | Per-molecular-subtype models (5 configs) |
| `sweep_test_peritumor.yaml` | Peritumor configuration sweep (12 configs) |
| `sweep_combat_matrix_*.yaml` | ComBat harmonization mode sweeps (4 modes each) |
| `sweep_covariates.yaml` | Covariate encoding comparison (4 configs) |

### Harmonization / ComBat
Available in `radiomics_train.py` and YAML configs:
- Modes: `none`, `zscore_site`, `combat_param`, `combat_nonparam`
- Harmonization is fold-safe in CV (fit on fold-train, apply to fold-val)
- Do not include `pcr` as a harmonization covariate

### Caching and Resume
Extraction uses per-patient checkpoint caching. Cache is invalidated by changes to image patterns, mask pattern, peritumor settings, force-2d, non-scalar handling, params YAML, or label override.

## Troubleshooting
- `mrmr` missing: install `mrmr-selection` in env
- `xgboost` missing: install `xgboost` if needed
- Label mismatch errors: verify `patient_id` consistency across labels/features/splits
- Empty class in fold: reduce `cv_folds` or adjust filtering
